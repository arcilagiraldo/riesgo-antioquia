🎯 ¿Qué es OMAIRA?

OMAIRA es una plataforma web de inteligencia artificial para la predicción, monitoreo y gestión del riesgo de desastres en el el territorio. Corre directamente en el navegador del operador — sin instalación, sin servidores propios — e integra en tiempo real datos de 17 fuentes institucionales y públicas para calcular un Índice de Riesgo Global (IRG) con credibilidad probabilística calibrada.

Principio fundamental: OMAIRA diferencia siempre entre dato real (proveniente de una fuente activa verificada) y estimado (calculado cuando la fuente no está disponible). Nunca presenta ficción como hecho. Cada variable indica su origen y nivel de confianza.

Población protegida: ~5.8 millones de habitantes · municipios activos · 9 subregiones · Marco Sendai 2015-2030.

🔬 Motor científico — cómo se calculan los riesgos

1. Índice de Riesgo Global (IRG)

IRG = Σ(wᵢ × vᵢ)   donde i = 1…23 variables
// wᵢ = pesos dinámicos ajustados por ENSO + estación climática
// vᵢ = variable normalizada en [0,1] — REAL si viene de fuente activa, ESTIMADO si no

2. Modelo de riesgo por tipo — 23 categorías

RIESGO = (H × E × V) × F_clima
// H = Amenaza: probabilidad del peligro físico (lluvia, sismo, temperatura...)
// E = Exposición: max(0.30, (pop/50.000)×0.35 + (pendiente/45)×0.65)
// V = Vulnerabilidad: (pendiente/45)×0.55 + (altitud/3000)×0.25 + 0.20
// F_clima = factor ENSO: ONI×0.04 + 1.04 (Niño→1.12, Niña→0.92, Neutro→1.04)

3. Variables clave del modelo

VariableFórmulaFuente principal
crec (creciente quebradas)ln×0.65 + hn×0.35 (sin embalse)Open-Meteo · SAMA · SIATA
API (memoria hidrológica)API_t = 0.85×API_{t-1} + P_t (Kohler-Linsley)IndexedDB acumulativo
inundación embalsemax(0, (embalse%-90)/10) × (1+lluvia×3) — solo si >90%EPM real · Proxy Open-Meteo

4. Memoria hidrológica acumulativa

El sistema guarda en IndexedDB los últimos 90 días de lluvia por municipio. Esto permite calcular el Antecedent Precipitation Index (API) real acumulado, la sequía por déficit real vs climatología normal de cada subregión, y mejorar la predicción de crecientes con suelo saturado. El modelo mejora automáticamente con el tiempo sin requerir intervención.

5. Prioridad de fuentes en los sensores

Cuando múltiples fuentes proveen el mismo dato (ej: lluvia), el sistema aplica jerarquía: SAMA (máxima — dato local calibrado) → SIATA (Área Metro) → IDEAM (ponderado 60/40 con Open-Meteo) → OpenWeatherMap (refinamiento) → Open-Meteo (base siempre disponible).

🎯 Sistema de credibilidad probabilística

OMAIRA implementa un sistema de evaluación de credibilidad basado en el estado del arte en machine learning probabilístico. Cada predicción incluye tres valores: Riesgo % (probabilidad calibrada del evento), Credibilidad % (confianza en esa predicción) e Incertidumbre (baja / media / alta / muy alta).

Métricas implementadas

MétricaFórmulaInterpretación
Brier ScoreBS = (1/N)×Σ(pᵢ - oᵢ)²0.00 = perfecto · 0.25 = sin información · Menor es mejor
Log LossLL = -(1/N)×Σ[o×log(p)+(1-o)×log(1-p)]Penaliza predicciones confiadas incorrectas · Menor es mejor
ECE (Calibration Error)ECE = Σ(n/N)×|acc-conf| por bin0 = perfectamente calibrado · Visualizado en Reliability Diagram
Isotonic Regression (PAVA)Pool Adjacent Violators AlgorithmCalibra probabilidades monotónicamente sin parámetros
Platt Scalingp_cal = σ(a×f+b) · gradient descent 200 iter.Calibración sigmoide paramétrica · Estima a,b con histórico

Credibilidad compuesta — 5 componentes

Cred = (Histórico×0.30 + Datos×0.20 + Fuentes×0.20 + Calibración×0.15 + Frescura×0.15) × Robustez
// Histórico: 1 - Brier×4 (mejora con cada outcome registrado)
// Datos: min(1, N_outcomes/200) — más datos = más confianza
// Fuentes: min(1, N_fuentes_activas/6)
// Calibración: 1 - ECE×5
// Frescura: 1.0 si <30min · 0.85 si <2h · 0.60 si >2h
// Robustez: penaliza si faltan fuentes críticas (Open-Meteo, USGS, NOAA)

Aprendizaje continuo (Online Learning)

El modelo actualiza las métricas incrementalmente con cada outcome registrado — sin recalcular todo desde cero. El peso de cada fuente en el ensemble se ajusta automáticamente según su Brier Skill Score histórico: peso = f(BSS × 0.5 + 0.5, n_muestras/200). Fuentes con mejor historial de predicción pesan más.

Para horizontes >72h, el modelo ajusta las probabilidades según la estación (temporadas de lluvias: marzo-mayo, septiembre-noviembre), la fase ENSO (El Niño amplifica sequía e incendios, La Niña amplifica lluvias y deslizamientos) y el tipo de riesgo.

🗺️ Cobertura territorial

OMAIRA cubre los municipios del territorio (de los 125 totales del departamento) organizados en 9 subregiones, con datos DANE 2026 verificados, altitudes corregidas y relieves calibrados. Cada municipio tiene configurados su fuente hídrica principal, riesgo predominante histórico y nivel de cobertura instrumental SAMA.

Los umbrales IDF (Intensidad-Duración-Frecuencia) para la activación de alertas están calibrados por subregión según metodología DAGRAN: Valle de Aburrá (28mm/1h), Oriente (35mm/1h), Urabá (48mm/1h), etc.
